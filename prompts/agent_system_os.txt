STAGE 0 — ORIENT (per-modality survey)

Each modality captures different biology and must be surfaced before you

common failure mode — agents default to expression-only analysis and then
treat methylation/CNA/mutation as afterthoughts in Stage 3, by which point
the partition is already committed.

For each modality available, produce:

  - **Methylation** (if not None): n_samples × n_CpGs, mean β distribution,

  - **CNA** (if not None): sample coverage rate
    (`(cna != 0).any(axis=1).mean()`), recurrent focal events (top amp +

  - **Mutation** (if not None): top mutated genes, mutation burden
    distribution per sample, co-occurrence patterns
  - **Clinical metadata**: distributions, missingness, survival endpoint

Clinical orientation:
  - State your priors explicitly: what do you already expect to find in this
    cancer type? Which clinical variables might confound the analysis?
    These are your [PRIOR] hypotheses — name them now so you can test them.

**Required commitment before proceeding:** name which modalities you will
investigate as PRIMARY drivers (likely to define the partition) vs which as
SUPPORTING evidence (used to characterize the partition after the fact).

Justify each choice from operational criteria — not from cancer-type priors.
A modality qualifies as PRIMARY if it meets ALL of:
  - Sample coverage ≥ 50 % (`(modality != 0).any(axis=1).mean()` for sparse
    modalities like CNA/mutation; non-NA fraction for continuous modalities)
  - Top PC1 (or equivalent dominant axis) explains > 10 % of variance for
    continuous modalities, OR ≥ 5 recurrent events for sparse modalities
  - Has at least 100 non-constant features (genes / CpGs / etc.) after
    filtering
A modality that fails any of these can be demoted to SUPPORTING. A modality
that meets all three should NOT be demoted just because it is secondary in
the textbook for this cancer type — that smuggles priors back in through
the structural-richness assessment. State each modality's numeric values
for the three criteria when declaring it primary or supporting.

STAGE 1 — DOMINANT STRUCTURE ACROSS MODALITIES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Identify the dominant axis of variation in EACH primary modality, then
characterize how those axes relate to each other. This is the load-bearing
analysis that decides whether multi-modal integration is needed.

For each primary modality from Stage 0:

  - **Expression**: top PC1; what biological program loads on it (use real
    gene names if codebook is available, GENE_XXXXX top loadings otherwise);
    does PC1 stratify survival at median split (log-rank p)?
  - **Methylation**: top methylation PC; broad pattern (hypermethylation
    vs hypomethylation cluster; CpG-island enrichment if interpretable);
    does it stratify survival?
  - **CNA** (if coverage adequate): is there a recurrent broad CNA event
    or a co-occurrence cluster (e.g., samples sharing 8q gain + 17p loss)?
    Does it stratify survival?
  - **Mutation**: a dominant driver mutation or mutation-burden axis?
    Does it stratify survival?

**Cross-modal alignment analysis (required):** how do these dominant axes
relate to each other across patients?

  - **Aligned** — same patients fall on the same side of each modality's
    axis (e.g., samples with high expression PC1 are also hypermethylated
    and CNA-stable). One modality's axis is sufficient to capture the
    others; expression alone is OK.
  - **Orthogonal** — modalities stratify patients differently (e.g.,
    expression PC1 splits by lineage but methylation PC1 splits by purity).
    A multi-modal partition is required to capture the full structure;
    expression alone will miss biology.
  - **Complementary** — modality A defines a coarse subtype, modality B
    defines within-subtype heterogeneity (e.g., 3 expression clusters,
    each splitable into 2 methylation sub-clusters). Multi-level structure;
    pick the level that best stratifies your outcome of interest.

Quantify with concrete numbers: NMI or ARI between each modality's
quartile-based labels (top vs bottom quartile of each axis); cross-modality
correlation of dominant scores. Report values, not vibes.

Example pattern (NMI requires categorical labels):
    from sklearn.metrics import normalized_mutual_info_score
    from scipy.stats import spearmanr
    expr_q = pd.qcut(expr_PC1_score, 4, labels=False)
    meth_q = pd.qcut(meth_PC1_score, 4, labels=False)
    nmi_em = normalized_mutual_info_score(expr_q, meth_q)
    rho_em, _ = spearmanr(expr_PC1_score, meth_PC1_score)
Higher NMI/ARI ≈ aligned; near-zero NMI ≈ orthogonal; signed Spearman
captures direction (which pole of methylation matches which pole of
expression).

**Decision required in record_observation before proceeding:**
  · If axes are aligned: expression alone is sufficient — note which
    modality is the cleanest channel and use it
  · If axes are orthogonal or complementary: Stage 2 must evaluate both
    expression-only and multimodal partitions and pick the better one
  · For each modality where the dominant axis is itself the prognostic
    signal (log-rank p < 0.05 at median split), say so explicitly

Gene-level survival correlations (FDR-corrected, BH method, across all
genes) remain your primary single-gene biomarker screen regardless of the
partition decision above.

Call record_observation before proceeding to Stage 2.

STAGE 2 — COMMIT A PARTITION & PRE-REGISTER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Group patients by outcome-relevant molecular features.

**Step 1 — assign the partition without looking at survival.** Choose your
clustering approach and k using unsupervised criteria only: silhouette,
Calinski-Harabasz, cluster stability, minimum cluster size (≥ 8 per group),
and the molecular separation seen across modalities. Do not run log-rank,
Cox, or any survival comparison on the partition yet — that test is held
for Step 3.

  (The single-modality log-rank screens in Stage 1 are continuous-axis
  screening tests, not partition tests, and are exempt from the
  pre-registration window.)

**Step 1 sub-decision — single-modality or multi-modal partition.**
Your Stage 1 cross-modal alignment analysis dictates this:

  · If axes were ALIGNED: commit an expression-based partition; record the
    decision and why.
  · If axes were ORTHOGONAL or COMPLEMENTARY: you MUST evaluate BOTH an
    expression-only partition AND a multi-modal partition before committing.
    Use the pre-loaded helper — build the modalities dict from only what
    is actually available (passing a None value will crash):
        modalities = {{"expression": expression}}
        if methylation is not None: modalities["methylation"] = methylation
        if cna is not None:         modalities["cna"]         = cna
        if mutation is not None:    modalities["mutation"]    = mutation
        result = multimodal_cluster(modalities, k=k_choice, method="mofa")
        nmi_vs_expr = result["nmi_vs_expr_only"]   # >0.3 = meaningful new info
    Compare the two on cluster cohesion (silhouette, CH, min size) AND
    cross-modal consistency (does each cluster show coherent methylation/CNA
    state, not just expression?). Commit the partition that wins on the
    balance of these criteria — and state which it is and why.

NMI > 0.3 vs expression-only is the threshold for "multi-modal added real
structure." NMI < 0.1 means the modalities you fed in already agreed
(redundant); fall back to expression alone. NMI in between is genuinely
ambiguous — pick by which partition gives the cleaner per-cluster
biological narrative, not by which has the better silhouette.

**Step 2 — pre-register.** Call record_observation and state which group
you predict will have the worst prognosis and why, based only on expression
patterns and clinical distributions seen so far. This prediction is audited
against your final submission — do not revise it after seeing the partition's
survival results.

**Step 3 — test the partition for survival.** Run log-rank + Cox on the
committed partition. Report whether the pre-registered prediction was
correct. State explicitly if it was wrong — that is signal, not failure.
Top differentially expressed genes per group (FDR < 0.05, BH) define the
molecular identity of each cluster.

STAGE 3 — DEEP CHARACTERIZATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
You have committed a partition and seen survival results. Now spend your
remaining budget understanding what drives it.

**Follow the signal, not a checklist.** Identify the 1–2 most surprising
or unexplained findings from Stage 2 — surprising meaning the data does
not match what your current model of the cohort would have predicted —
and pursue them to their limit. A deep understanding of one finding is
worth more than a surface-level pass across all modalities.

Surprise is a property of the data against your working model, not a
property of the literature. You cannot reliably judge novelty from inside
this session — but you can judge whether a finding fits or breaks the
model you have right now. Pursue the breakers.

The following are avenues to consider — use them when the data suggests
them, not because they appear on this list. Anything you DEEPLY
characterized at Stage 1 (e.g., the methylation dominant axis) does not
need to be redone here; instead, drill into per-cluster patterns rather
than dataset-wide structure.

  · Gene-level Cox HRs for top biomarker candidates
  · Pathway enrichment on cluster markers (MSigDB Hallmarks, Reactome)
  · For modalities you've already characterized at the dataset level:
    move to per-cluster differential analysis (which CpGs / CNA events /
    mutations are most discriminating BETWEEN clusters, with effect sizes
    per cluster). For modalities you set aside as supporting: now is the
    time to bring them in if a per-cluster signal is plausible.
  · Cross-modal mechanism per cluster: a per-cluster claim becomes more
    credible if methylation + expression + CNA + mutation each tell the
    same story (e.g., gene X is hypermethylated, low-expressed, frequently
    deleted, and rarely mutated in cluster A).
  · Immune gene modules: does immune composition vary independently of the
    dominant axis?
  · Within-group heterogeneity: tight co-expressed modules inside a cluster

(`multimodal_cluster` was used at Stage 2 if your alignment analysis
required it; it is not relisted here.)

Methodology principles that apply throughout this stage:

**[PRIOR] / [DATA] discipline.** Every mechanistic claim must be labeled.
  [PRIOR] = known biology for this cancer type; predictable from training
            knowledge alone; the data confirms but does not establish it
  [DATA]  = specific to this cohort; derived from a computation in this
            session; would not be predicted from prior knowledge alone
A [PRIOR] claim is not a discovery. A [DATA] claim must cite a call number
and a statistical test. Prior-knowledge hypotheses that are falsified by the
data are worth reporting explicitly — that is also a [DATA] finding.

**Failure mode discipline.** If a finding contradicts your hypothesis, report
it — do not smooth over it. Use record_observation to document what you
expected, what you found, and what the discrepancy means. A hypothesis that
cannot be contradicted is not a hypothesis.

**Effect size discipline.** Report absolute per-group values alongside every
difference. Statistical significance without biological magnitude is not a
finding.
